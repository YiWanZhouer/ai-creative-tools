### v1.8：发端引擎数据源（P0）

> 发言人在 03:10-04:32 花了最长时间讨论这一点。她不是在讨论创意生成器的"第一步选什么驱动力"，而是在讨论骨架生成器 L1 层的"叙事引擎从什么社会情绪切入"。**发端引擎的信号喂养是骨架生成器的事。**

#### 四维数据管道

```
L1 发端层 — 信号面板
  ├─ 热议话题（9 个月窗口）
  │   └─ 微博热搜 / 知乎热榜 / 百度指数
  ├─ 社交高赞（实时）
  │   └─ 小红书 / 抖音 / B站 高互动内容
  ├─ 电视剧题材（进行时 vs 完成时）
  │   └─ 区分"正在播/刚播完"和"去年播的"
  └─ 日韩前置信号（18 个月窗口）
      └─ 韩剧/日剧对社会性题材的发掘比国内早
```

#### 实施梯度

| 阶段 | 做法 | 依赖 |
|------|------|------|
| 轻量版 | 手动整理热点摘要 → 存入 L1 提示词模板 | 无 |
| 半自动 | 定时爬取 → 结构化 JSON → 喂入 | 爬虫 |
| 全自动 | 接入 API → 实时信号流 | API 接入 |

#### 与已有模块对接

- L1 卡片新增「📡 信号」按钮 → 点击展开当前热点摘要
- 类型片切换时，信号面板自动筛选相关信号（如恋综 → 情感话题优先）
- AI 脑暴可引用当前热点信号

### v2.6：决策者画像（P1）

发言人 09:19-10:34 的原话："甲方跟假方可是太不一样。他什么身份背景？财务出身的平台总监。有没有可能查到他过去的演讲学历？部门运营部门和商业化部门需求不一样。输进去，生成的是他给我的反向需求。"

**输入 → 输出链**：

```
决策者画像
  ├─ 所在部门：运营 / 商业化 / 内容 / 平台策略
  ├─ 背景身份：财务出身 / 制片出身 / 互联网产品出身 / 传统媒体出身
  └─ 可选：姓名 → 搜索公开演讲/文章
        ↓
AI 生成反向需求
  ├─ "这个人最可能问的 5 个问题"
  ├─ "这个人最可能反对的 3 个点"
  └─ "针对这个人的说服策略"
```
